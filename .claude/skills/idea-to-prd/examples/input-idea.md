# Raw Idea Notes

I want to build something that helps people book meetings by just CC'ing an assistant into email.

The problem is people hate Calendly links and it breaks the natural flow of email.

The assistant should take over the back and forth and work across time zones.

At first maybe Google Calendar only, maybe later Microsoft too.

It should work if two or three people are involved, not just one-on-one.

Need to be careful about privacy. I do not want to store message bodies if we can avoid it.

Would be nice if the assistant remembered preferences later, like preferred meeting times.

Not sure if this is MVP or later:
- multilingual support
- table booking for in-person meetings
- automatic follow-ups

Users are probably founders, operators, busy professionals.

Need to know:
- how much calendar access we actually need
- whether people trust an AI assistant in email
- what happens when attendees disagree on times
